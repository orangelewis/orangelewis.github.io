linux命令

echo

打印，字符串或系统变量
例子：echo $PATH
显示当前 PATH 环境变量，该变量的值由一系列以冒号分隔的目录名组成。
如：/usr/java/jdk1.8.0_121/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin

which

which 指令会在环境变量 $PATH 设置的目录里查找符合条件的文件；
安装完程序或者配置完变量后，最好的习惯是使用 which 看一下，检查一下环境是否配置正确，否则可能遇到自以为正确的 bug；
命令不存在，一般是环境变量没配置正确

source

通知当前 shell 读入路径为 filename 的文件并依次执行文件中的所有语句。
source 命令是 bash shell 的内置命令。而点命令（即 “.” 命令）则是 source 的另一名称。这两个命令都以一个脚本为参数，该脚本将作为当前 shell 的环境执行，即不会启动一个新的子进程。所有在脚本中设置的变量将成为当前 Shell 的一部分。

rz 和 sz

sz：将选定的文件发送（send）到本地机器
rz：运行该命令会弹出一个文件选择窗口，从本地选择文件上传到 Linux 服务器

alias

别名的使用可以简写冗长且难以记忆或者难以书写的命令
直接输入alias 查询已有别名

临时设置

‘’‘
alias cp=‘cp –i’
unalias cp 取消系统的别命令
’’’

永久

配置在环境变量中 /etc/profile ，~/.bashrc，~/.bash_profile 中，即永久配置别名

history

用于显示历史记录和执行过的指令命令。history 命令读取历史命令文件中的目录到历史命令缓冲区和将历史命令缓冲区中的目录写入命令文件。

history -c 命令可以清空当前窗口的历史输出命令。

快速使用

该命令单独使用时，仅显示历史命令，在命令行中，可以使用符号!执行指定序号的历史命令。例如，要执行第 2 个历史命令，则输入!2。

如何保存？

history 命令：是被保存在内存中的，当退出或者登录 shell 时，会自动保存或读取。在内存中，历史命令仅能够存储 1000 条历史命令，该数量是由环境变量 HISTSIZE进行控制。默认是不显示命令的执行时间，命令的执行时间，history 已经记录，只是没有显示。

查询历史命令

注：如想查询某个用户在系统上执行了什么命令，可以使用 root 用户身份登录系统，检查 Home 目录下的 用户主目录下的 “.bash_history” 文件，该文件记录了用户所使用的命令和历史信息。
彻底清空该文件夹的方式为：
cat /dell/null > .bash_history
cat .bash_history

rm

rm -f rz.log11 直接删除文件
rm -rf 1 直接删除文件夹

MAC与linux互传文件

文件夹比文件 多个-r
scp -r

mac 上传文件到 Linux 服务器

scp 文件名 用户名@服务器ip:目标路径
如：scp /Users/test/testFile test@www.linuxidc.com:/test/

Linux 服务器下载文件到 mac

scp 用户名@服务器ip:文件路径 目标路径
如：scp test@www.linuxidc.com:/test/testFile /Users/test/

环境变量

个人

个人环境变量配置在家目录下
~/.bash_profile
～/.bashrc 文件中
个人环境配置一些独自使用的程序变量，如果配置在用户的个人环境中，其他用户无法访问
PS:这里需要注意的是如果配置在～/.bash_profile 文件中，使用 ssh 远程连接的时候不会加载 ~/.bash_profile，造成一些无法排查的 bug

全局

全局环境变量的配置文件是：/etc/profile
全局环境变量中一般配置的是共用的程序环境
